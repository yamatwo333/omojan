# おもじゃん テスト計画書

## 1. この文書の目的

この文書は、`何をどう確認すれば MVP を完成とみなせるか` を整理するための文書です。

## 2. テストの対象

初版のテスト対象は次の通りです。

- ルーム作成 / 参加
- ロビー
- 3 ラウンドの提出進行
- 投票 / 再投票 / ホスト裁定
- 最終投票 / 総合優勝
- 長いワードの表示
- スマホブラウザ表示
- 基本的な AWS 公開動作

## 3. テスト方針

### 3-1. 初版で重視すること

- ルールどおり最後まで遊べること
- UI が面白さを邪魔しないこと
- 重大な進行不能がないこと
- 2〜4 人で破綻しないこと

### 3-2. テストレベル

- `単体確認`
  - 文字レイアウト、投票禁止、自動遷移条件など
- `画面確認`
  - 各画面の表示、活性 / 非活性、スマホ表示
- `結合確認`
  - ルーム作成から総合優勝までの通し確認
- `受け入れ確認`
  - 実際に身内で 1 試合遊んで違和感を洗い出す

## 4. 対応環境

### 優先確認

- iPhone Safari
- Android Chrome

### 補助確認

- デスクトップ Chrome

## 5. 重点テスト項目

### T01. ロビー / 参加

- 2 人、3 人、4 人で参加できる
- 表示名が正しく反映される
- 招待 URL コピーが動作する
- ホストが最初の順番を決められる
- 最近の優勝ワードが全体共通の履歴として表示される

### T02. ラウンド進行

- ラウンド 1 は 10 枚から 2 枚選べる
- ラウンド 2 では前ラウンドの使用牌が使えない
- ラウンド 3 ではさらに使用済み牌が非活性になる
- 順番制で手番が正しく進む

### T03. 提出 UI

- 2 枚選択ができる
- 順番入れ替えができる
- `ワードの境目で改行` が正しく働く
- `手動で改行` が 0 個でも使える
- `手動で改行` が複数箇所でも使える
- `1行で固定` が 1 行に収まる
- 長いワードでも極端に崩れない
- プレビューと提出後表示が大きくずれない
- 提出後は変更できない

### T04. 提出演出

- 提出時に全員の画面で中央ポップアップが出る
- 裏向きから表を向く演出が再生される
- ポップアップが画面外にはみ出さない
- スクロール位置に依存せず見える

### T05. 投票

- 自分のワードには投票できない
- 他人のワードには投票できる
- 投票後に待機画面へ移る
- 全員投票後に結果または再投票へ進む

### T06. 再投票 / ホスト裁定

- 同率時に再投票へ進む
- 再投票でも自分のワードに投票できない
- 再投票でも同率ならホスト裁定へ進む
- ホスト以外は裁定できない
- ホストは選択後に確定できる

### T07. ラウンド結果

- 勝利ワードが最優先で大きく表示される
- 票数内訳が見える
- 次ラウンド、最終投票への遷移が正しい
- 次ラウンド、最終投票への進行はホストだけが実行できる

### T08. 最終投票 / 総合優勝

- 各ラウンド勝利ワードが候補になる
- 自分のワードには投票できない
- 同率時の再投票、ホスト裁定が動く
- 総合優勝ポップアップが表示される
- 改行と書体が最終結果ポップアップまで保持される
- `もう一度最初から` はホストだけが実行できる

### T09. 履歴表示

- 最近の優勝ワードが 5 件表示される
- 一覧表示へ遷移できる
- 履歴が全体共通である

### T10. 再接続

- 同じ端末・同じブラウザから進行中ルームへ復帰できる
- 復帰後に自分の表示名とプレイヤー識別が維持される

### T11. デッキ管理

- 初版で使用するデッキマスターを運営側が更新できる
- 更新後のデッキ内容が反映される

### T12. 公開環境

- Amplify の公開 URL でトップが開く
- プロトタイプが開く
- 最新 push が自動デプロイされる

## 6. 代表テストケース

### 正常系

- TC-01: 4 人で 3 ラウンド + 最終投票まで通しで完了する
- TC-02: 2 人でも同じ流れで完了する
- TC-03: 3 人でも同じ流れで完了する

### 異常 / 境界系

- TC-04: 長いワードでも崩れずに提出できる
- TC-05: 手動改行なしでも長いワードが破綻しない
- TC-06: 同率 -> 再投票 -> ホスト裁定まで進める
- TC-07: 自分ワード投票不可がラウンド / 最終投票の両方で効く
- TC-08: 使用済み牌が再利用できない
- TC-09: 同じ端末から復帰して進行中ゲームへ戻れる

## 7. テスト実施の優先順位

### 優先度 A

- 進行不能がない
- 投票禁止条件が守られる
- 同率分岐が動く
- 長いワードの表示が崩れない

### 優先度 B

- スマホ表示の細かな見た目
- 待機画面の分かりやすさ
- 優勝ワード履歴
- 再接続の復帰体験

### 優先度 C

- デスクトップ表示の細部
- 補助的な導線の改善

## 8. 完了条件

MVP の完了条件は次の通りです。

- 優先度 A の項目に重大不具合がない
- 2〜4 人で通しプレイができる
- 総合優勝まで正常完了する
- 公開 URL で再現できる

## 9. テスト実施方法

### 開発中

- プロトタイプで随時ブラウザ確認
- 画面単位の確認
- 必要に応じて Playwright などで通し確認

### MVP 直前

- 実際に 2〜4 人でプレイして確認
- 面白さが減る UI 上の違和感を収集
- 進行テンポと待ち時間を確認
