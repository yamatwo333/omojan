# おもじゃん 要件定義書

## 1. この文書の目的

この文書は、`何を作るか` を決めるための文書です。

- 作る範囲
- MVP の対象
- 必要な機能
- 必要な品質
- 初版ではやらないこと

を整理します。

## 2. プロジェクト概要

- プロダクト名: `おもじゃん`
- 形態: スマホ向け Web ブラウザゲーム
- 想定利用: 招待 URL を友達に共有して、その場で 2〜4 人が対戦する
- 初版の目的: 少人数で問題なく 1 試合遊べること

## 3. MVP の定義

初版で目指すのは `2〜4人で、招待 URL から参加し、最後まで 1 試合遊べること` です。

MVP 完了条件は次の通りです。

- ロビー作成と参加ができる
- 最初の順番を決められる
- 3 ラウンドの提出と投票ができる
- 同率時の再投票とホスト裁定ができる
- 最終投票と総合優勝発表ができる
- 長めのワードでも提出 UI が破綻しない
- スマホブラウザで遊びやすい

## 4. 対象ユーザー

- 友達同士で短時間に笑えるゲームをしたい人
- アプリインストールなしで遊びたい人
- 大喜利やワードゲームが好きな人

## 5. 前提条件

- 参加人数は 2〜4 人
- 端末はスマホブラウザを優先する
- 招待 URL から参加する
- 初版ではログイン必須にしない
- 初版では固定デッキ 1 種でよい

## 6. 機能要件

### 6-1. ルーム機能

- FR-01: ルームを作成できる
- FR-02: 招待 URL でルームに参加できる
- FR-03: 表示名を入力して参加できる
- FR-04: ホストを識別できる
- FR-05: ロビーで参加人数と参加者一覧を確認できる
- FR-06: ホストが最初の順番を決められる
- FR-07: ロビーから最近の優勝ワードを確認できる
- FR-07-1: ロビーの優勝ワード履歴は `全体共通の履歴` を表示できる

### 6-2. ゲーム進行

- FR-08: ホストがゲーム開始できる
- FR-09: ゲーム開始時に各プレイヤーへ 10 枚の手札が配られる
- FR-10: 全 3 ラウンド進行できる
- FR-11: 各ラウンドで順番制の提出ができる
- FR-12: 使用済みの牌は次ラウンド以降使えない
- FR-13: ラウンド終了後に投票へ進める
- FR-14: 3 ラウンド終了後に最終投票へ進める
- FR-14-1: `次のラウンドへ` `最終投票へ` `もう一度最初から` はホストだけが実行できる

### 6-3. 造語提出 UI

- FR-15: 2 枚の牌を選択して造語を作れる
- FR-16: 牌の前後順を入れ替えられる
- FR-17: 改行モードを選べる
  - ワードの境目で改行
  - 手動で改行
  - 1行で固定
- FR-18: 手動改行は 0 個以上設定できる
- FR-19: 書体プリセットを選べる
- FR-20: プレビューと提出後表示が大きくずれない
- FR-21: 提出後は内容を変更できない
- FR-22: 提出時に全員の画面で中央ポップアップ表示される
- FR-22a: 提出ポップアップは全員が閉じるまで次の進行へ進まない
- FR-22b: 自分の手番画面でも、そのラウンドで公開済みの他プレイヤーのワードを確認できる

### 6-4. 投票・結果

- FR-23: 各ラウンドでワードに投票できる
- FR-24: 3人以上の対戦では自分のワードには投票できない
- FR-24a: 2人対戦では自分のワードにも投票できる
- FR-25: 同率時に再投票へ進める
- FR-26: 再投票でも同率時はホスト裁定へ進める
- FR-27: ラウンド結果で勝利ワードを大きく表示できる
- FR-28: 最終投票でも 3 人以上の対戦では自分のワードには投票できない
- FR-28a: 2人対戦の最終投票では自分のワードにも投票できる
- FR-29: 総合優勝ポップアップを表示できる
- FR-30: 最終結果一覧で総合優勝と各ラウンド勝利ワードを確認できる

### 6-5. 履歴表示

- FR-31: ロビーで最近の総合優勝ワードを最大 5 件表示できる
- FR-32: 優勝ワード一覧を別表示で確認できる
- FR-33: 優勝ワード履歴はルーム単位ではなく全体共通で保持できる

### 6-6. 再接続

- FR-34: 同じ端末・同じブラウザから再接続したときに、進行中のルームへ復帰できる
- FR-35: 復帰時に、自分の表示名とプレイヤー識別を再利用できる
- FR-35a: 同じブラウザ内で複数の招待ルームを並行して扱える

### 6-7. デッキ管理

- FR-36: 初版は 1 デッキ固定でよい
- FR-37: そのデッキの中身を運営側が編集・更新・削除できる
- FR-38: デッキ管理はプレイヤー向け UI でなくてもよい

## 7. 非機能要件

### 7-1. 利用環境

- NFR-01: スマホブラウザを最優先で対応する
- NFR-02: iPhone Safari と Android Chrome で遊べることを目標にする
- NFR-03: デスクトップでも確認はできるが、最適化対象はスマホ

### 7-2. UX

- NFR-04: 造語提出時の見せ場を最優先とする
- NFR-05: 1 画面に情報を詰め込みすぎない
- NFR-06: 結果画面では人名よりワードを優先表示する
- NFR-07: 長めのワードでも崩れず、かつ文字サイズを小さくしすぎない

### 7-3. 性能・同期

- NFR-08: 少人数対戦を前提にする
- NFR-09: 常時リアルタイム同期ではなくてもよい
- NFR-10: MVP では `操作時更新 + 待機画面での短周期取得` でよい

### 7-4. 運用・コスト

- NFR-11: 月額コストは小規模利用で数百円以内を目標とする
- NFR-12: 常時起動サーバーは避ける
- NFR-13: GitHub 連携で継続デプロイできること

### 7-5. セキュリティ

- NFR-14: 初版では簡易参加方式を採用する
- NFR-15: ルーム ID や招待 URL が第三者に推測されにくいこと
- NFR-16: root user の常用は避ける
- NFR-17: 同じ端末からの再接続で、プレイヤー識別を安定して復元できること

## 8. データ要件

- DR-01: デッキマスターを保持できる
- DR-01-1: デッキマスターは運営側が変更可能であること
- DR-02: ルーム状態を保持できる
- DR-03: プレイヤー一覧を保持できる
- DR-04: 各プレイヤーの初期手札を保持できる
- DR-05: 各ラウンドの提出内容を保持できる
- DR-06: 改行位置と書体を保持できる
- DR-07: 投票・再投票・ホスト裁定結果を保持できる
- DR-08: 最近の優勝ワード履歴を保持できる
- DR-09: 再接続のためのプレイヤー識別情報を保持できる

## 9. 初版の対象外

- 本格的なアカウント認証
- プレイヤー向けデッキ編集機能
- 複数デッキ選択
- チャット
- 観戦
- 通知
- ランキング
- 課金
- ホスト離脱時の複雑な権限移譲

## 10. 初版の前提方針

- 投票は匿名扱いとし、票数だけ表示する
- 制限時間は設けない
- リマッチは初版では必須にしない
- 2位以下の最終順位は初版では重視しない
- 再接続は `同じ端末・同じブラウザ` を対象にする
- デッキ管理は初版では運営側のメンテナンス方式でよい

## 11. 成果物

初版で必要な成果物は次の通りです。

- スマホ用フロントエンド
- 最小バックエンド API
- ルーム状態保存 DB
- 公開環境
- 開発ドキュメント
